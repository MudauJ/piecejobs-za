## 2. WHATSAPP JOB ALERTS FOR WORKERS
When a new job is posted, send WhatsApp notifications to all workers in the same city.

In the job posting submit function, after saving the job to Supabase:
1. Fetch all workers where city = job.city and is_verified = true
2. For each verified worker in that city, open a WhatsApp link... actually since we can't send bulk WhatsApp automatically without an API, instead:
   - Show a toast to the homeowner: "Your job has been posted! Verified workers in [city] will be notified."
   - Store a notifications_queue record for each worker

CREATE TABLE IF NOT EXISTS notifications_queue (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  worker_id UUID REFERENCES workers(id),
  job_id UUID REFERENCES jobs(id),
  message TEXT,
  status TEXT DEFAULT 'pending',
  created_at TIMESTAMPTZ DEFAULT NOW()
);
ALTER TABLE notifications_queue DISABLE ROW LEVEL SECURITY;
GRANT ALL ON notifications_queue TO anon, authenticated;

## 6. EMAIL NOTIFICATIONS
When key events happen, send an email using a simple mailto approach or log to a notifications table. For now, create an email_notifications table to queue emails:

CREATE TABLE IF NOT EXISTS email_notifications (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  to_email TEXT,
  subject TEXT,
  body TEXT,
  status TEXT DEFAULT 'pending',
  created_at TIMESTAMPTZ DEFAULT NOW()
);
ALTER TABLE email_notifications DISABLE ROW LEVEL SECURITY;
GRANT ALL ON email_notifications TO anon, authenticated;

Queue emails for these events:
- New application received → email to homeowner
- Application accepted → email to worker
- Payment received → email to worker
- Job completed → email to homeowner and worker

In the admin dashboard, add an "Emails" tab showing the email queue with status (pending/sent).
